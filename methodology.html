Methodology

Full details on the methodology behind our QUIC deployment analysis are available in our project repository. You can also browse the source code used for visualization.

Construction of the Dataset

We designed a multi-stage scanning pipeline to accurately identify and characterize QUIC deployment across the Vietnamese Internet. Our approach ensures comprehensive coverage by targeting the entire IPv4 address space allocated to Vietnam, while minimizing network intrusion through efficient scanning techniques.

The process begins with broad reachability testing and narrows down to specific protocol handshakes. We rely on RFC 9000 [RFC 9000] to verify compliance with the standardized IETF QUIC protocol, distinguishing it from legacy versions like gQUIC. This distinction is critical as early deployments [Zirngibl et al., 2021] often differ significantly from the final standard.

Properties of Tools & Data Sources

ZMap Scanner

Visit tool page

ZMap [Durumeric et al., 2013] is a fast single-packet network scanner designed for Internet-wide network surveys. We utilized ZMap to perform the initial discovery phase, scanning approximately 16 million IPv4 addresses.

The scan targets UDP port 443, the standard port for QUIC traffic. This step filters the vast address space down to a manageable list of potentially active hosts, which are then passed to subsequent stages for deeper analysis.

ZDNS

Visit tool page

Following the discovery of responsive IP addresses, we employed ZDNS to perform large-scale reverse DNS lookups. This tool allows us to map IP addresses to their domain names and infrastructure providers.

By resolving these addresses, we can categorize hosts into Internet Service Providers (ISPs), Content Delivery Networks (CDNs), or hosting services, providing context to the raw deployment numbers.

QScanner

Visit tool page

For the detailed protocol analysis, we deployed QScanner, a specialized tool developed for identifying QUIC deployments [Zirngibl et al., 2024]. Unlike simple port scanners, QScanner performs a full cryptographic handshake.

It negotiates QUIC versions, verifies TLS 1.3 parameters, and extracts certificate chains. This allows us to confirm that a server is not just listening on port 443 but is correctly speaking the QUIC protocol. We also use it to detect support for advanced features like Connection Migration [Buchet & Pelsser, 2025].

VNNIC Data

Visit provider page

Our target IP list is derived from the Vietnam Internet Network Information Center (VNNIC). We use their officially published statistics and IP allocation lists to define the boundaries of the Vietnamese Internet.

The dataset covers over 16 million IPv4 addresses. We cross-reference this with other ranking methodologies like Tranco [Le Pochat et al., 2019] and traffic reports [e-Conomy SEA 2024] to ensure our study reflects the most relevant and active portions of the local network ecosystem.
